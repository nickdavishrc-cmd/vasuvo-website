Key Data Flows

💊 Medication Reminder Flow

  1. User adds medication with reminder time in Android App
  2. Medication Service saves schedule to PostgreSQL
  3. Reminder Scheduler enqueues job in Bull/Redis Queue
  4. At scheduled time, Job Worker fires Notification Service
  5. Notification Service calls Google FCM → push to device
  6. If dose missed after grace period, escalation Level 1 triggers
  7. Level 1 → SMS to user; Level 2 → FCM + SMS to Care Partner; Level 3 → SOS alert

🔑 Authentication Flow

  1. User enters phone number on Android App
  2. Auth Service calls Firebase Auth / MSG91 to send OTP via SMS
  3. OTP stored in Redis with 5-minute TTL
  4. User submits OTP → Auth Service validates from Redis
  5. On success: JWT access token + refresh token issued
  6. User creates 4–6 digit PIN; stored as bcrypt hash in PostgreSQL
  7. Subsequent logins use PIN only; OTP only for re-registration

🤖 AI Insights Flow

  1. User opens AI Clinic; app sends request with JWT to API Gateway
  2. AI Service fetches user's medication list from PostgreSQL
  3. Checks Redis cache — if fresh insight exists, return immediately
  4. Cache miss: builds structured prompt with medications + conditions
  5. Calls Google Gemini API with safety-filtered prompt
  6. Response parsed, structured into categories (interactions, diet, lifestyle)
  7. Cached in Redis (24 h TTL); returned to app

📄 Medical Record Upload

  1. User selects photo / PDF in app
  2. App sends multipart upload to API Gateway
  3. Medical Records Service validates file type and size
  4. File uploaded to Google Cloud Storage (GCS) with encryption at rest
  5. Signed URL generated with short expiry for secure access
  6. Metadata (type, date, GCS path, owner ID) stored in MongoDB
  7. Care Partners with consent can access records via their own app

🆘 SOS Emergency Flow

  1. User taps SOS button (Care Plus only) and confirms
  2. App sends SOS event with current GPS location to API
  3. Activity Monitor Service processes emergency trigger
  4. Connections Service fetches all linked Care Partners
  5. FCM high-priority push sent simultaneously to all Care Partners
  6. SMS emergency alert dispatched to registered Emergency Contact
  7. Live location shared via WebSocket to all connected caregivers

💳 Subscription & Payment Flow

  1. User taps "Upgrade to Care Plus" in app
  2. Subscription Service creates Razorpay order via server-side API
  3. Order ID returned to app; Razorpay Android SDK launched
  4. User completes payment (UPI / Card / Net Banking)
  5. Razorpay sends payment webhook to Subscription Service
  6. Webhook signature verified (HMAC-SHA256) before processing
  7. Plan upgraded to Care Plus in PostgreSQL; features unlocked immediately

📍 Real-time Location Sharing

  1. Member enables location sharing in app
  2. App opens WebSocket connection to Location Service
  3. GPS coordinates pushed to server every 30 seconds
  4. Location Service publishes to member's Redis Pub/Sub channel
  5. Care Partner app subscribes to that channel via WebSocket
  6. Location updates streamed to Care Partner in real time
  7. Member disables → WebSocket closed, channel unsubscribed

📡 Activity Monitoring Flow

  1. App sends heartbeat ping to Activity Monitor every 15 minutes
  2. Last-active timestamp updated in Redis
  3. Cron job runs every 30 minutes to check all active members
  4. No activity for 4 h → MEDIUM alert sent to all Care Partners via FCM
  5. No activity for 6 h → HIGH alert; 8 h → EMERGENCY escalation
  6. SMS fallback sent at EMERGENCY level
  7. Member opens app → activity resolved; caregivers notified of resolution

🌐 Website Contact Form Flow

  1. Visitor fills form on vasuvo.com/care/contact.html
  2. Browser JS submits AJAX POST to Web3Forms API
  3. Web3Forms validates the access key server-side
  4. Honeypot field checked to block bots
  5. Email delivered to contact@vasuvo.com
  6. Success banner shown to visitor; form hidden
  7. No backend server involved (fully static hosting via GitHub Pages)

🔬 Lab Finder Flow

  1. User opens Lab Finder in app
  2. App requests device GPS location permission
  3. Location coordinates sent to Lab Finder Service
  4. Service queries Google Maps Places API for nearby labs
  5. Results filtered, sorted by distance, rated
  6. Lab list returned with distance, address, contact info
  7. User taps lab → Maps SDK opens for navigation directions

Security Architecture

🔐 Authentication & Authorization

  • OTP via SMS for registration (5-min Redis TTL)
  • 4–6 digit PIN for daily login (bcrypt hashed)
  • JWT access tokens (short TTL) + refresh tokens
  • Role-based access: Member / Care Partner / Professional
  • Per-connection explicit consent model for data sharing

🔒 Data Encryption

  • All traffic over TLS 1.3
  • Medical records encrypted at rest in GCS
  • Sensitive DB fields encrypted (AES-256)
  • Signed URLs for time-limited file access
  • Razorpay webhooks verified with HMAC-SHA256

🛡️ API Protection

  • Rate limiting per IP and per authenticated user
  • WAF for DDoS and SQL/code injection protection
  • CORS whitelist for allowed origins
  • All inputs validated and sanitised at boundary
  • Parameterised queries — no dynamic SQL

📋 Compliance & Audit

  • HIPAA-aligned data handling practices
  • Full audit trail for all data access and mutations
  • Member controls consent per Care Partner individually
  • Right to data deletion available in Profile settings
  • Regular dependency security audits

📱 Mobile Security

  • Certificate pinning to prevent MITM attacks
  • App PIN required on every session start
  • No sensitive data cached in plaintext on device
  • Biometric auth support (fingerprint / face unlock)
  • Remote session invalidation from server

☁️ Infrastructure Security

  • All backend services inside private VPC
  • Databases not exposed to public internet
  • Secrets via environment variables / HashiCorp Vault
  • Automated daily backups with point-in-time recovery
  • Least-privilege IAM roles for every service
